CS310 JGraphT Intro

 

Graph<V, E> API

 

 boolean addVertex(V v);

 boolean removeVertex(V v);

 boolean containsVertex(V v);

 Set<V> vertexSet();

 

 E addEdge(V sourceVertex, V targetVertex);  // let Graph gen. edge

 boolean addEdge(V sourceVertex, V targetVertex, E e); // app edge

 

 boolean containsEdge(V sourceVertex, V targetVertex);

 boolean containsEdge(E e);

 V getEdgeSource(E e);

 V getEdgeTarget(E e);

 double getEdgeWeight(E e);  // for weighted graphs, ignore for now

 Set<E> edgeSet();

 

 // edges for vertices:

 E getEdge(V sourceVertex, V targetVertex); // one of edges

 Set<E> getAllEdges(V sourceVertex, V targetVertex); // all the edges

 Set<E> edgesOf(V vertex);  // all edges touching vertex

 

 E removeEdge(V sourceVertex, V targetVertex);  // remove an edge

 boolean removeEdge(E e);  // remove the edge

 Set<E> removeAllEdges(V sourceVertex, V targetVertex);

 removeAllEdges(Collection<? extends E> edges);

   

These are all valid for directed and undirected graphs. Only a few of them need special interpretation for undirected graphs: In an undirected graph, there is (conceptually) an edge from one vertex to the other, but it stands for a symmetric connection.

containsEdge(V, V), getEdge(V,V): the qualifying edge can go from target to source

getEdgeSource, Target: the actual source and target (may be backwards to what you are expecting)

getAllEdges(V,V): some returned edges may go from target to source

removeAllEdges(V,V): removes ones connected the other way too

removeEdge(V,V): removes one which may be connected the other way

In other words, all these (V1,V2) arguments are working like ({V1,V2}), for an undirected graph.

 

For directed graphs, edgesOf(V) needs comment. It will return inbound edges as well as outbound, unlike what is described by an adjacency list (which only lists outbound edges).

 

In addition to the Graph interface type, there are subtypes DirectedGraph and UndirectedGraph:

 

Only for DirectedGraph<V, E>:

int inDegreeOf(V vertex);

Set<E> incomingEdgesOf(V vertex);

int outDegreeOf(V vertex);

Set<E> outgoingEdgesOf(V vertex);

 

Only for UndirectedGraph<V, E>:

int degreeOf(V vertex);

 

Constructors:

A simple graph is one with no loops and no more than one edge between any two vertices.

Example: a simple undirected Graph with String vertices and simplest possible edges:

Graph<String,BasicEdge> g =
      new SimpleGraph<String,BasicEdge>(BasicEdge.class);

 

The concrete class SimpleGraph implements a simple undirected graph as type UndirectedGraph, while SimpleDirectedGraph implements a simple directed graph as type DirectedGraph.  Note that a simple directed graph has no cycles of length 1 or length 2. For more general directed graphs, use DirectedMultigraph.

 

We could put:

 

UndirectedGraph<String,BasicEdge> g =
             new SimpleGraph<String,BasicEdge(BasicEdge.class);

if we wanted to use degreeOf(V) as well as all the Graph methods.

The  overall directory structure is something like games:

       jgrapht/src: Java sources

/javadoc: HTML API

/lib:       <--needed library:  jgraph.jar, for Swing graphics

 

Source packages under src:

   cs310: for pa5, pa6: LoadGraph.java, etc.

   cs310.visualization: support (glue code) for Swing graphics

   org.jgrapht: top-level package: Graph.java, etc.

   org.jgrapht.graph: implementation of graphs: SimpleGraph.java,
                                  SimpleDirectedGraph.java,...

   org.jgrapht.*: other implementation directories

top-level directory: CS310GraphIntro.html(this doc), build.xml, README.html, test.dat
